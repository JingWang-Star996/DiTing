## 🎯 核心教训（必须遵守）

### LL-001：必须使用清单式工作

> 永远使用检查清单，不依赖印象式工作

**违反后果**: 容易遗漏关键步骤，工作质量不稳定

**正确做法**: 创建详细的检查清单，逐项打勾确认

### LL-002：必须按岗位逐一审查

> 每个岗位、每个成员都必须被审查到，不能跳过

**违反后果**: 容易遗漏某些岗位，注意力偏差导致不公平

**正确做法**: 列出完整成员名单，按岗位分类，逐个审查

### LL-003：必须分块读取避免数据截断

> 大数据必须分块读取，不能一次性读取

**违反后果**: 数据被截断，部分数据丢失

**正确做法**: 每次读取 200 行，检查 truncated 标志，循环读取直到完成

### LL-004：必须交叉验证完整性

> 写完必须验证，交叉检查，不能直接提交

**违反后果**: 错误无法及时发现，需要返工修正

**正确做法**: 创建验证清单，交叉检查，验证通过后再提交

### LL-005：岗位平等原则

> 程序、策划、美术同等重要，不能偏废

**违反后果**: 某些岗位被忽视，团队士气受损

**正确做法**: 平等对待每个岗位，给予同等关注

## 🎯 使用场景

### 场景 1：周报汇总

```
1. 读取日报数据（分块读取，避免截断）
2. 审查每个成员的工作（按岗位逐一审查）
3. 按岗位汇总（程序/策划/美术）
4. 填写质量指标（覆盖率/完整率/准确率）
5. 验证完整性（交叉检查，确认无遗漏）
```

### 场景 2：事故复盘

```
1. 记录事故详情（5W1H 描述）
2. 进行 5 Why 分析（找出根本原因）
3. 制定改进措施（短期/中期/长期）
4. 追踪执行情况（定期检查，闭环管理）
5. 更新知识库（案例/最佳实践/经验教训）
```

### 场景 3：质量审计

```
1. 日常检查（每次流程执行后 24 小时内）
2. 定期审计（每月一次，全面检查）
3. 专项审计（发生重大问题时）
4. 审计报告（评分 + 问题 + 改进建议）
5. 整改跟踪（直到问题关闭）
```

**创建时间**：2026-04-08 19:40
**触发事件**：制作人总结遗漏美术组工作内容教训
**优先级**：最高

---

## 📌 问题描述

**事件**：2026-04-08 周报汇总时，制作人总结遗漏了美术组（罗园超、汤世军、曾军）的工作内容。

**影响**：
- 周报不完整，无法全面反映团队工作进展
- 美术组工作成果未被记录
- 需要返工修正

---

## 🔍 根本原因分析

### 1. 数据读取不完整
- 读取日报表格时返回 210 行数据，但被截断（truncated: true）
- 只处理了前 200 行左右的数据，部分数据在截断区域

### 2. 汇总流程缺陷
- ❌ 没有先列出 2 组完整成员名单（11 人）
- ❌ 没有按岗位逐一审查每个成员的日报
- ❌ 写完总结后没有交叉验证

### 3. 注意力偏差
- 程序组（4 人）：工作内容多且复杂 → 重点关注
- 策划组（3 人）：重点项目 → 重点关注
- 美术组（3 人）：工作内容相对单一 → 容易被忽略

### 4. 缺乏系统性方法
- 没有建立标准流程：成员名单→按岗分类→逐一汇总→完整性验证
- 依赖"印象式"汇总，而非"清单式"汇总

#### 2. 读取日报数据（分块读取）
```
步骤：
1. 第一次读取：A1:G200（前 200 行）
2. 检查 truncated 标志：
   - 如果 truncated=false → 数据完整，继续下一步
   - 如果 truncated=true → 继续读取下一块
3. 第二次读取：A201:G400（第 201-400 行）
4. 第三次读取：A401:G600（如需要）
5. 合并所有数据块
6. 按日期范围筛选数据
7. 按成员姓名分组整理

技术实现：
```javascript
// 分块读取示例
let allData = [];
let offset = 1;
let hasMore = true;

while (hasMore) {
    const endRow = offset + 199;
    const range = `A${offset}:G${endRow}`;
    const result = await feishu_sheet.read({ range, url });
    
    allData.push(...result.values);
    
    // 检查是否还有更多数据
    hasMore = result.truncated === true || result.values.length === 200;
    offset += 200;
}

// 验证数据完整性
console.log(`总计读取 ${allData.length} 行数据`);
```

检查点：
- [ ] truncated 标志是否为 false
- [ ] 最后几行是否为空行（确认读到末尾）
- [ ] 日期范围是否正确
- [ ] 所有成员数据是否都有
```

## 🛡️ 异常处理机制（2026-04-08 QA 部门新增）

### 1. 数据读取异常
- **现象**：truncated=true 持续多次 / API 报错
- **处理**：
  a. 重试机制（最多 3 次，间隔 5 秒）
  b. 降级方案（手动导出 CSV）
  c. 上报机制（通知负责人）
- **记录**：记录异常原因、处理过程、结果

### 2. 数据不一致
- **现象**：汇总数据与原始数据不符
- **处理**：
  a. 立即停止写入
  b. 重新读取验证
  c. 人工核对关键数据
  d. 确认无误后继续
- **记录**：记录不一致内容、原因分析、处理结果

### 3. 流程中断
- **现象**：中途中断（系统故障/人员离开）
- **处理**：
  a. 断点记录（记录已完成步骤）
  b. 恢复机制（从断点继续）
  c. 验证机制（恢复后验证完整性）
- **记录**：记录中断原因、断点位置、恢复结果

### 4. 回滚机制
- **触发条件**：发现严重错误
- **回滚步骤**：
  a. 备份当前状态
  b. 恢复到上一个正确版本
  c. 分析问题原因
  d. 修正后重新执行
- **记录**：记录回滚原因、回滚版本、修正内容

## 📝 代码模式

### 标准流程示例
```javascript
// 1. 获取成员名单
const members = await getTeamMembers();
const group2Members = filterByGroup(members, '2 组');
const byRole = groupByRole(group2Members);

// 2. 读取日报数据（分块读取避免截断）
const dailyReports = [];
let offset = 1;
let hasMore = true;
while (hasMore) {
    const batch = await readDailyReports(offset, 100);
    dailyReports.push(...batch);
    hasMore = batch.length === 100;
    offset += 100;
}

// 3. 按岗位逐一汇总
const summary = {
    program: summarizeByRole(dailyReports, byRole.program),
    planning: summarizeByRole(dailyReports, byRole.planning),
    art: summarizeByRole(dailyReports, byRole.art)
};

// 4. 完整性验证
validateSummary(summary, byRole);

// 5. 写入周报
await writeWeeklyReport(summary);
```

## 🎯 核心原则

1. **清单式工作**：永远使用检查清单，不依赖印象
2. **逐一审查**：每个成员都必须被审查到
3. **完整性验证**：写完必须验证，交叉检查
4. **分块读取**：大数据分块读取，避免截断
5. **岗位平等**：程序、策划、美术同等重要

## 🎯 系统定位

**质量体系 + 复盘体系** 一体化平台

| 模块 | 职责 |
|------|------|
| **质量体系** | 质量标准、检测、把关、告警 |
| **复盘体系** | 事故分析、经验沉淀、持续改进 |

## 📋 核心功能

### 1. 质量体系

```
需求评审 → 代码审查 → 测试验证 → 发布检查 → 线上监控
   ↓          ↓          ↓          ↓          ↓
质量标准   代码规范   测试用例   发布清单   监控告警
```

### 2. 复盘体系

```
事故发生 → 信息收集 → 根因分析 → 改进措施 → 效果追踪
   ↓          ↓          ↓          ↓          ↓
事故报告   时间线    5Why 分析   行动项     验证关闭
```

## 🚀 快速开始

### 创建质量检查

```bash
# 创建新的质量检查项
heimdall quality create --name "代码审查" --type "review"
```

### 创建复盘记录

```bash
# 创建新的复盘记录
heimdall retrospective create --incident "事故 ID" --date "2026-04-08"
```

### 生成质量报告

```bash
# 生成本周质量报告
heimdall report weekly --output "report.md"
```

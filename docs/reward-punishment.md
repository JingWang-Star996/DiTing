## 🎯 一、总则

### 1.1 目的

为激励员工严格执行标准流程，确保工作质量，预防问题复发，特制定本奖惩机制。

### 1.2 原则

- **公平公正**: 奖惩标准公开透明，一视同仁
- **奖罚分明**: 奖励优秀，惩罚违规，不姑息不偏袒
- **及时有效**: 奖惩及时执行，确保激励效果
- **教育为主**: 惩罚不是目的，改进才是目标

### 1.3 适用范围

- **适用对象**: 全体执行人员、部门负责人、QA 部门人员
- **适用场景**: 周报汇总、问题复盘、流程执行、知识管理等质量相关工作

## 🏆 二、奖励机制

### 2.1 奖励类型

#### 类型 1：质量优秀奖

| 项目 | 内容 |
|------|------|
| **评选条件** | 连续 3 个月审计评分≥90 分（优秀） |
| **评选频率** | 每季度一次 |
| **奖励对象** | 执行人员、团队 |
| **奖励内容** | 通报表扬 + 绩效加分 + 物质奖励 |

**评分标准**:
- 流程执行合规性：≥27 分（满分 30 分）
- 质量度量指标达成：≥27 分（满分 30 分）
- 异常处理记录完整性：≥18 分（满分 20 分）
- 知识管理更新情况：≥18 分（满分 20 分）
- 总分：≥90 分

**奖励标准**:

| 连续优秀月数 | 奖励内容 |
|------------|---------|
| 3 个月 | 通报表扬 + 绩效 +5 分 |
| 6 个月 | 通报表扬 + 绩效 +10 分 + 奖金 300 元 |
| 12 个月 | 通报表扬 + 绩效 +20 分 + 奖金 1000 元 + 年度优秀员工候选 |

#### 类型 2：零差错奖

| 项目 | 内容 |
|------|------|
| **评选条件** | 连续 3 个月无差错（无问题复发、无质量事故） |
| **评选频率** | 每季度一次 |
| **奖励对象** | 执行人员 |
| **奖励内容** | 通报表扬 + 绩效加分 |

**差错定义**:
- 流程执行错误（跳过步骤、未按标准执行）
- 数据错误（数据不准确、数据遗漏）
- 质量事故（造成重大影响的问题）
- 问题复发（已解决的问题再次发生）

#### 类型 3：改进贡献奖

| 项目 | 内容 |
|------|------|
| **评选条件** | 提出优秀改进建议并被采纳，产生显著效果 |
| **评选频率** | 按需评选（建议被采纳后） |
| **奖励对象** | 全体员工 |
| **奖励内容** | 通报表扬 + 绩效加分 + 物质奖励 |

**评选标准**:

| 改进效果 | 奖励内容 |
|---------|---------|
| 小改进（效率提升 10% 以内） | 通报表扬 + 绩效 +2 分 |
| 中改进（效率提升 10-30%） | 通报表扬 + 绩效 +5 分 + 奖金 200 元 |
| 大改进（效率提升 30% 以上） | 通报表扬 + 绩效 +10 分 + 奖金 500 元 |
| 重大改进（流程变革、成本大幅降低） | 通报表扬 + 绩效 +20 分 + 奖金 1000 元 + 年度优秀员工候选 |

#### 类型 4：知识贡献奖

| 项目 | 内容 |
|------|------|
| **评选条件** | 在知识管理方面有突出贡献（案例编写、最佳实践沉淀、经验分享） |
| **评选频率** | 每季度一次 |
| **奖励对象** | 全体员工 |
| **奖励内容** | 通报表扬 + 绩效加分 + 物质奖励 |

**评选标准**:

| 贡献类型 | 数量要求 | 奖励内容 |
|---------|---------|---------|
| 案例编写 | 1 个高质量案例 | 通报表扬 + 绩效 +2 分 |
| 最佳实践沉淀 | 1 个最佳实践 | 通报表扬 + 绩效 +3 分 + 奖金 100 元 |
| 经验分享 | 1 次高质量分享 | 通报表扬 + 绩效 +2 分 |
| 培训教材编写 | 1 份培训材料 | 通报表扬 + 绩效 +5 分 + 奖金 200 元 |

#### 类型 5：年度质量奖

| 项目 | 内容 |
|------|------|
| **评选条件** | 年度审计平均分≥90 分，且无重大质量问题 |
| **评选频率** | 每年一次 |
| **奖励对象** | 执行人员、团队 |
| **奖励内容** | 年度优秀员工候选 + 奖金 2000-5000 元 + 荣誉证书 |

### 2.2 奖励流程

```
提名 → 审核 → 批准 → 公示 → 实施 → 归档
 ↓      ↓      ↓      ↓      ↓      ↓
QA 部门  部门负责人  公司管理层  全员  执行奖励  记录档案
```

**流程说明**:
1. **提名**（QA 部门）: 每季度/年度结束后 5 个工作日内
2. **审核**（部门负责人）: 收到提名后 3 个工作日内
3. **批准**（公司管理层）: 收到审核后 3 个工作日内
4. **公示**（全员）: 批准后 2 个工作日内，公示期 3 个工作日
5. **实施**（人力资源部门）: 公示期结束后 5 个工作日内
6. **归档**（QA 部门 + 人力资源部门）: 实施完成后 3 个工作日内

## 🚨 三、惩罚机制

### 3.1 惩罚类型

#### 类型 1：流程违规

| 违规级别 | 违规描述 | 惩罚措施 |
|---------|---------|---------|
| 轻微违规 | 偶尔忘记填写检查清单（1 个月内≤2 次） | 口头提醒 + 补填 |
| 一般违规 | 经常忘记填写检查清单（1 个月内 3-5 次） | 通报批评 + 重新培训 |
| 严重违规 | 故意跳过流程步骤（1 个月内≥3 次） | 通报批评 + 绩效 -5 分 |
| 重大违规 | 故意违反流程造成质量问题 | 严重警告 + 绩效 -10 分 + 调岗考虑 |

#### 类型 2：质量问题

| 问题级别 | 问题描述 | 惩罚措施 |
|---------|---------|---------|
| 轻微问题 | 数据小错误（不影响整体质量） | 口头提醒 + 修正 |
| 一般问题 | 数据遗漏/错误（需要返工） | 通报批评 + 绩效 -2 分 |
| 严重问题 | 重大数据错误（影响决策） | 通报批评 + 绩效 -5 分 + 重新培训 |
| 重大问题 | 质量问题造成重大损失 | 严重警告 + 绩效 -10 分 + 调岗考虑 |

#### 类型 3：问题复发

| 复发级别 | 复发描述 | 惩罚措施 |
|---------|---------|---------|
| 首次复发 | 已解决的问题再次发生（1 次） | 通报批评 + 绩效 -3 分 |
| 多次复发 | 同样问题复发≥2 次 | 严重警告 + 绩效 -5 分 + 流程再造 |
| 系统性复发 | 系统性问题导致多次复发 | 部门负责人问责 + 绩效 -10 分 |

### 3.2 惩罚流程

```
发现问题 → 调查核实 → 确定级别 → 批准 → 执行 → 归档
   ↓          ↓          ↓          ↓      ↓      ↓
QA 部门    QA 部门    QA 部门    管理层  人力资源  记录档案
```

**流程说明**:
1. **发现问题**（QA 部门）: 审计/检查/举报发现
2. **调查核实**（QA 部门）: 收集证据，访谈相关人员
3. **确定级别**（QA 部门）: 根据标准确定惩罚级别
4. **批准**（管理层）: 批准惩罚方案
5. **执行**（人力资源部门）: 执行惩罚（通报/扣分等）
6. **归档**（QA 部门 + 人力资源部门）: 记录惩罚信息

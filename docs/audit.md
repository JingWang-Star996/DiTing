# 质量审计流程 - 质量管理知识库

**创建日期**: 2026-04-08  
**维护部门**: 质量管理部门（QA）  
**文档状态**: 🟢 已启用  
**适用场景**: 周报汇总、问题复盘、流程执行审查

---

## 📋 一、审计目标

1. **确保流程执行合规**: 验证各项标准流程是否被正确执行
2. **确保质量指标达标**: 验证质量度量指标是否达到目标值
3. **发现问题并改进**: 识别流程执行中的问题，推动持续改进
4. **预防问题复发**: 通过审计防止已解决的问题再次发生

---

## 🔍 二、审计类型

### 2.1 日常检查

| 项目 | 内容 |
|------|------|
| **检查人** | 制作人 / 部门负责人 |
| **检查频率** | 每次流程执行后 |
| **检查时间** | 流程完成后 24 小时内 |
| **检查方式** | 抽查验证 |

**检查内容**:
- [ ] 检查清单是否完整填写
- [ ] 质量度量指标是否填写
- [ ] 异常情况是否记录（如有）
- [ ] 知识库是否更新（如有新问题/最佳实践）

**检查方法**:
1. 随机抽查 3-5 个检查点
2. 验证检查清单填写完整性
3. 核实质量度量数据准确性
4. 记录检查结果

---

### 2.2 定期审计

| 项目 | 内容 |
|------|------|
| **审计人** | QA 部门 |
| **审计频率** | 每月一次 |
| **审计时间** | 每月最后一周 |
| **审计范围** | 当月所有流程执行记录 |
| **审计时长** | 1-2 个工作日 |

**审计内容**:

#### 审计项目 1：流程执行合规性

| 审计要点 | 审计方法 | 合格标准 |
|---------|---------|---------|
| 是否按标准流程执行 | 检查检查清单填写情况 | 清单完整率 100% |
| 是否有跳过步骤 | 对比标准流程与实际执行记录 | 无跳过步骤 |
| 是否有未授权变更 | 检查流程变更记录 | 所有变更都有审批 |

#### 审计项目 2：质量度量指标达成

| 审计要点 | 审计方法 | 合格标准 |
|---------|---------|---------|
| 成员覆盖率 | 检查质量度量表格 | ≥100% |
| 数据完整率 | 检查质量度量表格 | ≥100% |
| 数据准确率 | 抽查验证原始数据 | ≥98% |
| 按时完成率 | 检查完成时间记录 | ≥95% |

#### 审计项目 3：异常处理记录完整性

| 审计要点 | 审计方法 | 合格标准 |
|---------|---------|---------|
| 异常是否完整记录 | 检查异常记录表格 | 记录完整率 100% |
| 异常处理是否规范 | 检查处理流程记录 | 符合异常处理流程 |
| 根本原因分析是否深入 | 检查 5 Why 分析 | 分析到根本原因 |
| 改进措施是否落实 | 检查改进计划执行 | 措施完成率 100% |

#### 审计项目 4：知识管理更新情况

| 审计要点 | 审计方法 | 合格标准 |
|---------|---------|---------|
| 案例库是否更新 | 检查知识库更新记录 | 新问题 100% 入库 |
| 最佳实践是否沉淀 | 检查最佳实践文档 | 优秀做法及时沉淀 |
| 经验教训是否记录 | 检查经验教训库 | 教训 100% 记录 |
| 培训教材是否更新 | 检查培训材料 | 及时纳入新内容 |

**审计评分标准**:

| 评分等级 | 分数范围 | 说明 |
|---------|---------|------|
| 优秀 | 90-100 分 | 所有项目达标，无重大问题 |
| 良好 | 80-89 分 | 大部分项目达标，有小问题 |
| 合格 | 70-79 分 | 基本达标，需要改进 |
| 不合格 | <70 分 | 多项未达标，需要立即整改 |

**评分计算**:
- 流程执行合规性：30 分
- 质量度量指标达成：30 分
- 异常处理记录完整性：20 分
- 知识管理更新情况：20 分
- 总分：100 分

---

### 2.3 专项审计

| 项目 | 内容 |
|------|------|
| **触发条件** | 发生重大问题 / 重复发生同样问题 / 管理层要求 |
| **审计人** | QA 部门 + 相关部门 |
| **审计时间** | 问题发生后 48 小时内 |
| **审计范围** | 问题相关的所有流程和数据 |

**审计流程**:
1. 问题识别：明确审计范围和目标
2. 数据收集：收集相关流程记录、数据、文档
3. 现场审计：访谈相关人员，查看实际操作
4. 问题分析：分析根本原因和系统缺陷
5. 审计报告：出具审计报告和改进建议
6. 整改跟踪：跟踪整改措施执行情况
